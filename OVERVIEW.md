# Обзор проекта: P2P-платформа для аренды автомобилей

## 📋 Описание проекта

Данное веб-приложение представляет собой P2P-платформу для аренды автомобилей между частными лицами. Система разработана как Single Page Application (SPA) без использования `react-router`, что обеспечивает быструю навигацию между разделами без перезагрузки страницы.

**Основная цель проекта**: Создание удобной платформы для прямой аренды автомобилей между пользователями с возможностью управления автопарком, бронированиями и отзывами.

## 🎯 Основные возможности системы

### Для всех пользователей (включая неавторизованных):

1. **Просмотр каталога автомобилей**
   - Фильтрация по типу топлива, коробке передач
   - Поиск по марке и модели
   - Отображение информации об автомобилях

2. **Просмотр аренд**
   - Просмотр списка активных аренд
   - Фильтрация по статусу

3. **Просмотр локаций**
   - Список доступных точек выдачи
   - Информация о локациях

### Для авторизованных пользователей (Владелец):

1. **Управление автомобилями**
   - Добавление своих автомобилей
   - Редактирование информации
   - Изменение статуса доступности

2. **Управление арендами**
   - Подтверждение и отклонение заявок
   - Отслеживание статуса аренд

### Для администраторов:

Все функции владельца, плюс:

1. **Управление автомобилями** (CRUD операции)
   - Добавление и редактирование любых авто
   - Деактивация автомобилей
   - Полный контроль над каталогом

2. **Управление локациями** (CRUD операции)
   - Создание и редактирование локаций
   - Деактивация локаций

3. **Управление арендами** (CRUD операции)
   - Полный доступ ко всем арендам
   - Изменение статусов

## 🔐 Система ролей и прав доступа

### Роли пользователей:

1. **guest** (Гость) - роль по умолчанию
   - Просмотр каталога авто, аренд, локаций
   - Нет возможности редактирования данных

2. **owner** (Владелец) - требует авторизацию
   - Все права гостя
   - Управление своими автомобилями
   - Управление своими арендами

3. **admin** (Администратор) - требует авторизацию
   - Все права владельца
   - Управление всеми автомобилями (CRUD)
   - Управление локациями (CRUD)
   - Доступ к разделу "Администрирование"

**Пароли для входа** находятся в файле `CREDENTIALS.md`

## 🗄 Структура данных

### Основные сущности:

1. **Car (Автомобиль)**
   - id, brand, model, year
   - fuelType, transmission
   - pricePerDay, status
   - ownerId, locationId, isActive

2. **Rental (Аренда)**
   - id, carId, renterId
   - startDate, endDate
   - totalPrice, status
   - notes

3. **Location (Локация)**
   - id, name, address
   - city, coordinates
   - isActive

4. **Review (Отзыв)**
   - id, carId, userId
   - rating, comment
   - date

## 📚 Что желательно понимать для защиты диплома

### Теоретическая часть:

1. **SPA (Single Page Application)**
   - Что такое SPA и его преимущества? Ответ: SPA - это веб-приложение, которое загружается один раз, а затем все переходы между страницами происходят без перезагрузки. Преимущества: быстрая работа, плавные переходы, лучший UX.
   - Как реализована навигация? Ответ: Через NavigationStore (MobX), который переключает компоненты в App.tsx.

2. **State Management (MobX)**
   - Зачем нужен MobX? Ответ: Для централизованного управления состоянием приложения.
   - Как организованы stores? Ответ: AuthStore (авторизация), DataStore (данные), NavigationStore (навигация), UIStore (UI).

3. **TypeScript**
   - Преимущества типизации? Ответ: Обнаружение ошибок на этапе компиляции.

4. **Firebase Realtime Database**
   - Что такое Firebase? Ответ: Облачная NoSQL база данных от Google.

5. **P2P модель**
   - Что такое P2P? Ответ: Peer-to-peer - модель, где пользователи взаимодействуют напрямую друг с другом.
